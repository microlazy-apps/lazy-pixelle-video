# lazy-pixelle-video — project context for Claude Code

懒猫微服 (LazyCat MicroServer) lpk wrapper for [Pixelle-Video](https://github.com/AIDC-AI/Pixelle-Video).
Upstream Pixelle-Video is vendored as a git subtree at `vendor/pixelle-video/`.
**All adaptations live in `patches/`** — vendor/pixelle-video stays pristine.

Build/publish/bootstrap logic is **not** in this repo — it lives in
[microlazy-apps/lazycat-ci](https://github.com/microlazy-apps/lazycat-ci).

## Architecture: single Streamlit process

Upstream Pixelle-Video ships as two Python services:
- `web/` — Streamlit Web UI (port 8501) used by humans
- `api/` — FastAPI backend (port 8000) for SDK / MCP integration

**Both directly import the `pixelle_video` library**; web does NOT call api
over HTTP. The api process is therefore optional in a "Web UI for human users"
deployment. We only run Streamlit; api is deliberately not started.

`patches/01-lazycat-entrypoint.patch` adds `lazycat-entrypoint.sh` and
overrides the upstream Dockerfile's `CMD` with our own `ENTRYPOINT`. The
entrypoint:

1. Seeds `${PERSIST_DIR}/config.yaml` from `config.example.yaml` if missing.
2. Symlinks `/app/config.yaml -> ${PERSIST_DIR}/config.yaml`.
3. Exec's Streamlit on `:8501`.

## No deploy-params; settings live in the Web UI

We deliberately do NOT expose deploy params for LLM key / ComfyUI URL /
RunningHub key. The upstream Streamlit Sidebar already has a Settings panel
(`web/components/settings.py`) that writes back to the same `config.yaml`
through `pixelle_video.config.manager`. Asking for these once at install time
and again in the UI is redundant; users edit them once in the UI and the
lazycat bind keeps them across restarts.

If you ever need install-time params, re-introduce `lazycat/lzc-deploy-params.yml`
and add the `deploy-params:` line back to `lzc-build.yml`. The entrypoint
script's history (`patches/01-lazycat-entrypoint.patch` in v0.1.0 of this
repo) showed how to render env -> config.yaml on first start.

## Persistent volumes

- `/lzcapp/var/persist:/app/persist` — config.yaml (Web UI settings)
- `/lzcapp/var/data:/app/data` — user-uploaded BGM / templates / workflows /
  per-user resources (the upstream `data/*` override layer)
- `/lzcapp/var/output:/app/output` — generated videos

The `/app/temp` dir from upstream Dockerfile is intentionally NOT bind-mounted
— transient working files are fine to lose on container restart.

## Streamlit behind lazycat reverse proxy

Lazycat HTTP routes/upstreams downgrade to HTTP/1.1, which is fine for
Streamlit (it uses HTTP/1.1 + WebSocket Upgrade for the runtime channel).
Required env vars (set in `lzc-manifest.template.yml`):

- `STREAMLIT_SERVER_HEADLESS=true` — don't open browser at start
- `STREAMLIT_SERVER_ENABLE_XSRF_PROTECTION=false` — accept reverse-proxied origin
- `STREAMLIT_SERVER_ENABLE_CORS=false` — same reason
- `STREAMLIT_SERVER_ADDRESS=0.0.0.0` — listen on all interfaces
- `STREAMLIT_SERVER_PORT=8501` — match the upstream/manifest port

Health endpoint is Streamlit's built-in `/_stcore/health` (200 OK during
runtime, used by lazycat's `health_check.test_url`).

## Patches workflow

```sh
# Apply patches into vendor for live editing:
git apply patches/01-lazycat-entrypoint.patch -p1 --directory=vendor/pixelle-video

# Make changes...
vim vendor/pixelle-video/lazycat-entrypoint.sh

# Stage new files so git diff sees them:
git add -N vendor/pixelle-video/<any-new-files>

# Regenerate the patch:
git diff --no-color --relative=vendor/pixelle-video \
  vendor/pixelle-video/ > patches/01-lazycat-entrypoint.patch

# Restore vendor for clean commit (only patches/* should be tracked):
git checkout HEAD -- vendor/pixelle-video/Dockerfile
rm vendor/pixelle-video/<new-files>
git reset HEAD vendor/pixelle-video/<new-files>

# Verify:
git apply --check patches/01-lazycat-entrypoint.patch \
  -p1 --directory=vendor/pixelle-video
```

CI applies patches in lexical order (`patches/*.patch`) inside lazycat-ci's
docker job before `docker build`, with `--directory=vendor/pixelle-video`.

## Release flow (`v*` tag)

```
git tag v0.1.0 && git push --tags
  ↓
release.yml (caller, this repo)
  ├── prep:    strip leading v -> 0.1.0
  ├── build:   uses lazycat-ci/lpk-build.yml@v0.1.0
  │             apply patches/* to vendor/pixelle-video/
  │             docker (ubuntu) -> push ghcr (~3-4GB image: chromium+ffmpeg+CJK fonts)
  │             build  (xiaomoru) -> copy-image -> render templates ->
  │                                   lzc-cli project build -> artifact:lpk
  └── publish: uses lazycat-ci/publish.yml@v0.1.0
               publish-release (ubuntu) -> GH Release upload .lpk
               publish-appstore (xiaomoru) -> lzc-cli appstore publish
```

Image is large (~3-4 GB) because the upstream Dockerfile installs:
- Chromium + system deps via `playwright install --with-deps chromium`
- ffmpeg + fonts-noto-cjk
- Python venv with pixelle-video and all transitive deps

There's no obvious way to slim it without breaking video rendering (Playwright
is used to render HTML templates to PNG frames).

## Bootstrap (first-time appstore submission)

`bootstrap-app.yml` is `workflow_dispatch` with `version` (required) +
`app_id` (optional, skip `/app/create` on retry) inputs. Calls
`lazycat-ci/lpk-build.yml` then `lazycat-ci/bootstrap.yml`.

- `/app/create` returns 500 on duplicate package — pass `app_id` workflow
  input to skip create on retry.
- If a previous review record is in pending state, `/app/{id}/review/create`
  returns 500 "record not found". Workaround: delete the app in the lazycat
  developer center, then re-run with no `app_id`.

- Org self-hosted runner `xiaomoru` registered at microlazy-apps level (done)
- Org/repo secrets `LAZYCAT_USERNAME`, `LAZYCAT_PASSWORD` (done)
- ghcr package `microlazy-apps/lazy-pixelle-video` visibility — defaults to
  PRIVATE on first push, must be flipped to Public at
  https://github.com/orgs/microlazy-apps/packages/container/lazy-pixelle-video/settings
  (Danger Zone) before lazycat's `copy-image` can pull it.
- Lazycat developer center registration of `cloud.lazycat.app.pixelle-video`
  — done via `bootstrap-app.yml`.

## Updating Pixelle-Video upstream

```sh
git subtree pull --prefix=vendor/pixelle-video \
  https://github.com/AIDC-AI/Pixelle-Video.git main --squash

# Re-verify the patch still applies:
git apply --check patches/01-lazycat-entrypoint.patch \
  -p1 --directory=vendor/pixelle-video
```

If upstream changes Streamlit entry path, config-loader behavior, or the
Dockerfile's `CMD`/`COPY` layout, edit the patch (re-derive via the patches
workflow above) before tagging a new release.

## Local lpk build (smoke test)

Requires `lzc-cli`, `envsubst` (gettext), and docker on PATH.

```sh
# one-time
git clone https://github.com/microlazy-apps/lazycat-ci ~/lazycat-ci

# in this repo:
bash ~/lazycat-ci/scripts/build-lpk.sh \
  --version 0.0.0-dev \
  --docker-context ./vendor/pixelle-video \
  --patches-target vendor/pixelle-video
# → lazycat/cloud.lazycat.app.pixelle-video-0.0.0-dev.lpk
```

## Versioning notes

- Upstream AIDC-AI/Pixelle-Video uses `v0.x.y` (current upstream: 0.1.15).
- This wrapper picks its own semver starting at `v0.1.0`.
- Bump version on every meaningful upstream pull or patch change.
